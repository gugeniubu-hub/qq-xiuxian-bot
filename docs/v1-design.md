# V1 Design Notes

## 产品定位

这是一个适合 QQ 群游玩的轻量文字修仙游戏。

它不是:

- 复杂管理型公会游戏
- 靠网页交互驱动的单人挂机页游
- 纯随机抽卡数值器

它应该是:

- 玩家通过 QQ 机器人发指令操作
- 每天多次短回合互动
- 群体活跃能反哺个人收益
- 中后期通过轮回与隐藏内容延长寿命

## 参考项目结论

从 `react-xiuxian-game` 借鉴:

- 境界、世界观、事件表达
- 单人养成的情绪反馈

从 NoneBot 修仙插件借鉴:

- 指令流
- 群聊持久化
- 坊市与图片消息经验

明确不继承的部分:

- 重宗门建设
- 冗长职位体系
- 过多功能入口

## 初版闭环

### 新手 10 分钟体验

1. `入道`
2. `签到`
3. `历练`
4. 触发一次 `奇遇`
5. 查看 `我的状态`
6. 尝试 `闭关`
7. 凑够修为后第一次 `突破`

### 中期 3-7 天体验

1. 学第一本宗门功法
2. 开始参与坊市交易
3. 用残篇或材料换取资源
4. 追求第一次大境界圆满

### 长期体验

1. 轮回解锁隐藏内容
2. 追求不同 build
3. 参与世界事件和限时黑市

## 轮回设计

轮回不是简单 prestige，而是内容钥匙。

### 核心资源

- `轮回次数`: 身份深度
- `轮回印记`: 用于启动或强化轮回
- `前尘点`: 用于解锁永久天赋

### 永久解锁方向

1. 隐藏宗门
2. 特殊命格
3. 古老功法
4. 更高灵根保底
5. 稀有奇遇池
6. 坊市特权

### 不建议的做法

1. 轮回后保留太多数值  
   会很快碾压新玩家。

2. 轮回只给倍率  
   那会显得很空。

3. 所有好内容都塞给轮回  
   新人会失去乐趣。

## 宗门设计

每个宗门只定义三类东西:

1. 主题
2. 功法路线
3. 入门条件

示例:

- 青岚宗: 稳定吐纳，突破加成低，适合新手
- 赤霄门: 历练收益高，奇遇风险更大
- 太虚观: 一转后解锁，福缘与命格相关

## 经济设计

### 灵石流入

- 签到
- 历练
- 奇遇
- 出售物品

### 灵石流出

- 突破准备
- 购买材料
- 坊市交易手续费
- 黑市高价商品
- 转世材料

### 为什么要福缘池

它能让交易税不只是扣钱，而是回流到群体活跃上。

## 风控与刷屏

初版从一开始就要考虑:

1. 长文本尽量转图片
2. 高频指令带冷却
3. 单次返回别太长
4. 每日收益有软上限
