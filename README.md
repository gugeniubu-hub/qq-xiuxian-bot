# qxianbot

一个面向 QQ 群聊的文字修仙游戏机器人。

项目目标不是复刻大型修仙插件，而是做一个适合群友长期游玩的轻量修仙 MMO:

- 指令短，反馈快，尽量不刷屏
- 以群聊为主界面，由 QQ 机器人响应玩家操作
- 宗门只提供功法与传承路线，不做重工会建设玩法
- 坊市以玩家互相交易为核心，手续费进入公共福缘池
- 奇遇、历练、突破带有明确的 `roll` 随机性
- 每日天象会影响历练、闭关与奇遇权重
- 转世之外，再加入寿元压力、灵根属性、人物属性、地图探索、功法熟练度、冲关底蕴、炼丹与斗法这几条长线

## 当前阶段

这是当前可开群测试的 V1 版本，包含:

- 项目结构与运行方式
- V1 玩法设计
- 转世重修与长期解锁设计
- SQLite 数据模型
- NoneBot2 + OneBot V11 机器人入口
- 可试玩的初始命令循环

当前已接入:

- 入道 [道号] / 我的状态 / 属性 / 重骰灵根 / 地图 / 探索 / 签到 / 天象
- 闭关 / 出关
- 历练 / 奇遇 / 突破 / 斗法 / 古藏试炼
- 服用 / 参悟 / 炼丹
- 宗门列表 / 加入宗门 / 我的功法 / 功法详情 / 请法 / 主修功法
- 背包 / 我的法宝 / 装备法宝
- 坊市 / 坊市上架 / 坊市购买
- 世界事件 / 领取事件
- 排行 / 转世 / 命格 / 最近记录 / 新手引导

当前这版还补了几项更适合群测的体验层能力:

- 状态面板会给出“下一步建议”
- 面板会展示气血、物攻、法攻、物防、法防、速度、悟性、道悟、破境、丹道等人物属性
- 地图探索会按境界与转世解锁，并联动灵根相性、功法、法宝与人物属性
- 历练会触发野怪、劫修、古阵、星坠裂隙等事件，可能受伤或濒死，也可能领悟野外稀有功法
- 提供最近几次行动记录，方便回看历练 / 奇遇 / 斗法结果
- 高活跃主动玩法支持可配置轻量冷却，避免群聊被短时间刷屏
- 丹药效果已收口为更直观的单用途设计

## QQ 机器人接入方式

本项目现在同时兼容两种接入:

1. 官方 QQ 开放平台创建的机器人，推荐用于你的场景。
2. NapCat / OneBot V11，作为兼容路径保留。

### 官方 QQ 开放平台机器人

推荐优先使用 WebSocket 模式。服务启动后会主动连接 QQ 网关，不需要先配置公网回调地址。

环境变量示例:

```dotenv
DRIVER=~fastapi+~httpx+~websockets
HOST=0.0.0.0
PORT=8080

QQ_IS_SANDBOX=false
QQ_VERIFY_WEBHOOK=true
QQ_BOTS='[{"id":"你的AppID","token":"你的Token","secret":"你的AppSecret","intent":{"c2c_group_at_messages":true,"interaction":true},"use_websocket":true}]'
```

如果你想使用 Webhook 模式，则把 `use_websocket` 改成 `false`，并在 QQ 开放平台配置 HTTPS 回调地址:

```text
https://你的域名/qxian/qq/webhook
```

公域 QQ 群机器人通常会把玩家标识为 `member_openid`，不是传统 QQ 号。因此群里斗法建议用:

```text
斗法 @目标
```

而不是手输 QQ 号。项目会继续兼容 OneBot 的 `斗法 QQ号`。

### OneBot / NapCat

如果使用 NapCat 或其他 OneBot V11 实现，保留下面配置:

```dotenv
ONEBOT_ACCESS_TOKEN=
ONEBOT_SECRET=
```

### 常用短指令

`COMMAND_START=["","/"]` 时，中文指令和斜杠短指令都能用。常用例子:

```text
/help    帮助
/start   新手引导
/me      我的状态
/stats   人物属性
/map     地图
/explore 青岚山
/daily   签到
/bag     背包
/market  坊市
/rank    排行
/pk @目标 斗法
/world   天象
/event   世界事件
/claim   领取事件
```

### QQ 按钮菜单

官方 QQ 机器人支持 Markdown 消息按钮后，项目会在帮助、面板、地图、背包、功法、历练结果等回复下附带快捷按钮。常用按钮会走回调事件，玩家点击后直接执行，例如:

```text
面板 / 签到 / 历练 / 地图 / 奇遇 / 背包 / 功法 / 排行
```

复杂命令仍保留文字输入，例如 `斗法 @目标`、`坊市上架 物品名 单价 数量`、`主修功法 功法名`。如果按钮权限暂未生效，可设置:

```dotenv
QXIAN_QQ_BUTTONS_ENABLED=false
```

关闭按钮层后，原文字指令不受影响。

## 为什么这样做

参考了这些项目:

- [JeasonLoop/react-xiuxian-game](https://github.com/JeasonLoop/react-xiuxian-game)
- [s52047qwas/nonebot_plugin_xiuxian](https://github.com/s52047qwas/nonebot_plugin_xiuxian)
- [ganchengcaiyue/nonebot_plugin_simulator_xiuxian](https://github.com/ganchengcaiyue/nonebot_plugin_simulator_xiuxian)

得到的结论是:

1. 网页修仙项目适合借鉴世界观、境界和事件表达。
2. QQ 群机器人项目更值得借鉴的是指令流、持久化、风控和多人节奏。
3. 初版不能一上来就做成超重系统，否则指令负担和维护成本都会很快失控。

## V1 设计

### 核心循环

`入道 -> 面板/属性 -> 地图探索 -> 历练/奇遇 -> 功法成长 -> 炼丹 / 斗法 -> 突破 -> 市场流通 -> 转世再构筑`

### V1 系统范围

1. 角色系统
2. 境界与突破
3. 吐纳挂机
4. 历练与奇遇
5. 宗门与功法
6. 丹药、炼丹与材料循环
7. 斗法与轻 PK
8. 每日天象与寿元压力
9. 坊市交易与福缘池
10. 每日签到与基础排行榜
11. 转世重修与前尘解锁

### 角色系统

角色字段:

- QQ 号
- 道号
- 灵根
- 境界
- 修为
- 年龄
- 寿元
- 灵石
- 福缘
- 体力
- 悟性
- 轮回次数
- 轮回印记
- 前尘点
- 当前宗门
- 已学功法
- 功法熟练度
- 背包

灵根决定修炼倍率，初版做五档:

- 凡灵根
- 黄灵根
- 玄灵根
- 地灵根
- 天灵根

并进一步拆成:

- 主属性: 金 / 木 / 水 / 火 / 土 / 风 / 雷 / 冰
- 纯度: 影响修炼、悟道、突破上限
- 性情与特质: 作为轻量词条保留，影响少量随机倾向

对玩家展示时会刻意简化为“某属性灵根 + 关键加成”:

- 金: 物攻、法宝、破防
- 木: 修炼、气血、灵材
- 水: 悟道、炼丹、续航
- 火: 历练、爆发、冲关
- 土: 物防、稳固、突破
- 风: 速度、奇遇、探索
- 雷: 斗法、爆发、天关
- 冰: 法攻、法防、控场

人物属性为衍生值，不单独存库，会随境界、修为、灵根、功法、熟练度、法宝、命格与转世实时变化:

- 气血: 斗法承伤与高风险地图容错
- 物攻: 剑修、体修、法宝类斗法压制
- 法攻: 法修、雷火冰系招式和术法爆发
- 物防: 体修承伤和历练受挫损耗
- 法防: 术法抗性、冰水系续航和斗法稳定性
- 速度: 先手、奇遇、抢机缘与地图风险规避
- 悟性: 修炼、参悟和部分地图收益
- 道悟: 参悟、古藏、炼丹稳定性与突破加成
- 破境: 冲关成功率与破境型地图收益
- 丹道: 炼丹方向与药材地图收益

### 境界与突破

初版境界:

1. 炼气
2. 筑基
3. 金丹
4. 元婴
5. 化神

每个大境界拆成前期 / 中期 / 后期 / 圆满。

突破逻辑:

- 每个小境界需要修为门槛
- 满足门槛后可执行 `突破`
- 突破时进行一次或多次 `roll`
- 基础成功率由当前境界决定
- 功法、福缘、丹药、奇遇状态可修正成功率
- 失败会损失部分修为，并进入短暂冷却

这样既保留随机性，也能留下养成空间。

### 寿元与天象

这版开始把“时间”也纳入循环：

- 历练、奇遇、闭关会缓慢推进年龄
- 寿元越逼近极限，收益越会衰减，突破也会受压制
- `天象` 指令可查看当天世界状态
- 部分稀有道具如 `延寿果` 可以提供少量延寿空间

这让群友在后期不只是无脑堆数值，而是会自然开始考虑：

- 今天适不适合闭关
- 要不要拿悟道札记去参悟已学功法
- 是继续冲境界，还是先补寿元和资源

### 转世重修

这是 V1 里最重要的中长期系统之一。

设计目的:

- 给老玩家提供新的追求，不只是无穷冲数值
- 让部分稀有内容有明确门槛
- 让“练新号”变成角色成长的一部分

转世开启条件建议:

1. 达到 `化神圆满`
2. 完成一次渡劫型事件
3. 拥有足够的 `轮回印记` 或特殊材料

转世效果:

- 境界重置回 `炼气前期`
- 修为清空
- 保留部分非数值成长
- 获得 `前尘点`
- 累积 `轮回次数`

转世后的永久收益:

- 更高品质灵根保底
- 解锁隐藏宗门
- 解锁禁术 / 古传承 / 命格
- 增加奇遇高段位事件权重
- 解锁更高阶坊市权限或特殊交易位
- 解锁 `洗髓丹` 这类需要轮回体质承受的深层构筑内容

建议首批轮回解锁:

1. `一转`: 解锁隐藏宗门与轮回功法
2. `一转`: 解锁洗髓丹与第一次灵根重塑路线
3. `二转`: 解锁命格系统与特殊奇遇支线
4. `三转`: 解锁禁术、古遗迹、上古交易人

这样就能形成:

`日常变强 -> 首次化神 -> 转世 -> 解锁新内容 -> 再次构筑角色`

这条线会让游戏寿命长很多，也让转世真正“有必要但不强迫”:

- 不转世，依然能完整体验基础修炼闭环
- 想做更极致的灵根、功法、斗法 build，就必须转世
- 高阶宗门、轮回功法、洗髓、稀有奇遇因此都有了明确门槛

### 吐纳挂机

命令:

- `入道`
- `闭关`
- `出关`
- `我的状态`

设计原则:

- 群里不能要求玩家一直在线
- 挂机收益要稳定，主动玩法收益要略高

闭关提供稳定修为增长:

- 与闭关时长相关
- 受灵根、境界、主修功法加成
- 设收益上限，避免无限挂

当前闭关模式:

- `闭关 [分钟] 吐纳`: 稳定拿修为
- `闭关 [分钟] 凝练`: 修为略低，但更容易攒冲关底蕴
- `闭关 [分钟] 参玄`: 修为更少，但道悟和功法成长更高
- `闭关 [分钟] 冲关`: 专门为大境界突破做准备

### 历练与奇遇

命令:

- `历练`
- `奇遇`
- `地图`
- `探索 地图名`

历练与地图探索是主动玩法主入口:

- 消耗体力，当前版本已降低体力消耗并加快自然恢复
- 获得修为、灵石、材料，低概率获得悟道札记
- 触发野怪、劫修、古阵、星坠裂隙等历练事件，可能受伤、濒死或获得额外收获
- 极低概率直接领悟完整野外稀有功法，如 `玄霜封魂录`、`万兽炼体经`、`星陨剑典`、`风雷遁甲篇`

地图探索会提供更明确的目标:

- `青岚山`: 入门修炼与基础灵材
- `沧浪泽`: 炼丹材料与道悟
- `赤霞岭`: 冲关底蕴与火性材料
- `玄铁谷`: 法宝线索、悟道札记与灵石
- `乱风原`: 奇遇、福缘与悟道札记
- `雷泽`: 一转后逐步解锁，偏轮回印记、洗髓玉、雷系法宝
- `太虚古域`: 二转后逐步解锁，偏悟道札记、洗髓与后期构筑

奇遇采用显式随机:

- 机器人先给出事件文本
- 系统执行 `d100` 或多段 `roll`
- 根据点数区间决定结果

示例:

- `1-20`: 遭遇反噬，损失体力或修为
- `21-60`: 平稳收获，得到普通奖励
- `61-90`: 上佳奇遇，得到额外灵石或材料
- `91-100`: 大机缘，得到野外稀有功法 / 高品质丹药 / 福缘提升

这样玩家对随机结果有感知，不会像黑箱。

### 丹药与炼丹

当前版本把丹药补成了完整闭环，而不是只掉落后直接吃掉:

- `历练 / 奇遇` 会掉落灵草、灵泉水、赤焰砂、月华粉、洗髓玉
- `炼丹 丹药名` 进行显式 `roll`
- 成功时可能一炉双丹，失败会产出 `丹渣`
- 炼丹成功率会被悟性、道悟、主修功法风格、灵根属性、天象共同影响

当前可炼丹药:

- `聚气丹`: 稳定修为来源
- `回灵散`: 回复体力，给主动玩法续航
- `凝元丹`: 增加 `冲关底蕴`
- `悟道丹`: 增长 `道悟`
- `洗髓丹`: 一转后开放，用来洗练灵根纯度、性情、特质，低概率偏转主属性

这样丹药系统同时承担三件事:

1. 日常消耗口  
2. 冲关准备资源  
3. 转世后的 build 微调器

### 斗法与 PK

当前版本已经接入轻量斗法:

- `斗法 @目标`
- `斗法 QQ号`

设计原则:

- 显式展示双方 `roll`
- 展示双方总势，胜负不黑箱
- 胜者获得少量灵石、修为，优势明显时额外得一点道悟
- 败者只损失少量修为与体力，不过分打击
- 灵根、功法类型、主修风格、熟练度、冲关底蕴、轮回次数都会影响结果

这样 `战诀`、`锻体法`、轮回流和高熟练主修，都开始真正影响玩家之间的互动体验。

### 宗门与功法

这部分按你的要求收紧:

- 宗门不是重工会系统
- 宗门不做建设、职位管理、资材升级那一整套
- 宗门的核心意义是传承和功法获取

V1 宗门玩法:

1. 玩家可加入一个宗门
2. 每个宗门有自己的功法池
3. 宗门提供主修功法和辅修法门
4. 宗门声望达到条件后可领取更高阶功法
5. 宗门每日可触发一次师门任务，奖励宗门声望和随机功法感悟
6. 部分宗门需要达到指定轮回次数才能加入

宗门只保留三个数值:

- 宗门名
- 宗门传承路线
- 玩家个人宗门声望

这样够用，而且不会变成另一个管理游戏。

本版又向前推进了一步:

- 功法有品阶 / 类型 / 属性 / 风格
- 玩家可以显式切换 `主修功法`
- 功法与灵根属性存在相性
- 突破后会自动感应并承接宗门的后续传承

### 坊市与福缘池

坊市坚持“玩家交易为主”:

- 玩家可以上架背包物品
- 其他玩家可以买走
- 每笔成交收取固定比例手续费
- 手续费进入 `福缘池`

福缘池发放规则:

1. 每日签到时，从福缘池中拿出固定比例
2. 在当天签到玩家里随机分配
3. 高福缘玩家获得大奖概率略高，但不能完全碾压

这样坊市不是单纯抽税，而是会反哺群体活跃。

推荐初版规则:

- 手续费 5%
- 每日发放池子余额的 20%
- 单次签到基础奖励 + 福缘池抽奖奖励
- 轮回后可解锁更高档位上架数量或特殊拍品类型

### 每日签到

命令:

- `签到`

签到奖励:

- 固定灵石
- 固定少量修为
- 一次福缘池随机分红
- 极低概率触发“今日鸿运”状态

签到的意义不是送钱，而是提醒回流和激活群聊。

### 排行榜

命令:

- `排行`
- `境界排行`
- `灵石排行`
- `福缘排行`

初版先支持文本榜单，后续补图片榜单。

### 其他值得一起带上的玩法

为了让群里玩起来不空，我建议 V1 顺手把这几条也预留进去:

1. 寿元与老去  
   高境界玩家有压迫感，转世才更有意义。

2. 命格词条  
   比如“逢凶化吉”“财运亨通”“劫后余生”，作为轻量 build 系统。

3. 世界事件  
   每天固定时段刷 1 次全群事件，大家一起 `应劫`、`护山`、`夺宝`。

4. 悟道札记
   历练掉落札记，用于参悟已学功法和提升熟练，不承担新功法合成。

5. 黑市 / 游商  
   与普通坊市错开，提供限时高价收购或特殊道具，是灵石回收口。

6. 斗法榜 / 连胜气势  
   不做重竞技，但可以让群里自然形成“谁最近手感火热”的讨论。

7. 丹方稀有度  
   让高阶丹方本身也能成为交易物，继续抬高坊市深度。

### 可选 AI 说书人

可接入 DeepSeek 等 OpenAI 兼容接口做文案增强。默认关闭，模型不可用、超时、未配置 Key 或额度不足时，游戏会直接使用本地原本文案，不影响数值判定、奖励发放、PK 胜负、坊市交易和数据库写入。

建议先只开启文案层:

```env
QXIAN_LLM_ENABLED=true
QXIAN_LLM_PROVIDER=deepseek
QXIAN_LLM_BASE_URL=https://api.deepseek.com
QXIAN_LLM_MODEL=deepseek-v4-flash
QXIAN_LLM_API_KEY=你的 API Key
QXIAN_LLM_TIMEOUT_SECONDS=3
QXIAN_LLM_DAILY_LIMIT=300
```

当前 AI 只用于:

- 历练描述润色
- 斗法回合战报润色
- 未识别 `/xx` 指令的智能提示
